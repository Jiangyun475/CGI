目前主流多模态领域的对比学习，几乎全是“拉近模态 A 和模态 B（如 CLIP）”或者“拉近正样本，推开负样本”。我们要设计一种完全没人用过、专门为你这个 Ortho 架构量身定制的对比学习。1. 核心哲学重构：不要对比“实体”，去对比“交互的灵魂”在药物-基因互作中，真正决定活性的不是药物长什么样，也不是基因长什么样，而是它们相遇时产生的“化学反应”模式。在你的网络中，$V_g$ 和 $V_{c\perp}$ 已经被你强制正交化了（即 $V_g \cdot V_{c\perp} = 0$）。这意味着，如果我们将它们做哈达玛积（元素级相乘）：$$H_{sig} = V_g \odot V_{c\perp}$$这个 $H_{sig}$ 向量代表了纯粹的、非线性的“协同交互激活模式”（Interaction Signature）。并且由于它们正交，这个 $H_{sig}$ 向量所有分量的和绝对等于 0。这在流形空间中是一个极其完美的约束平面。2. 全新创新点：同类交互签名对齐 (SupCon on Interaction Signatures)传统做法： 把药物 A 和基因 A 拉近。我们的做法： 药物 A 和基因 A 结合成功了（标签 1），药物 B 和基因 B 也结合成功了（标签 1）。虽然药物 A 和 B 截然不同，基因 A 和 B 也截然不同，但它们成功的“协同交互模式（$H_{sig}$）”应该具有相似的数学形态。机制： 我们在 Batch 内，让所有真实发生的互作（标签 1）的 $H_{sig}$ 互相靠近；让所有无效互作（标签 0）的 $H_{sig}$ 互相远离。这在物理意义上极度优雅：网络被逼迫着去学习一种通用的“结合范式”，而不是死记硬背具体的分子结构。四、 代码实现：替换旧的 CL 模块你可以直接用下面这个类替换掉你代码中的 FocalDirectionAwareCL。不需要改变模型输出，也不需要改变训练循环（除了传入参数的微调）。Pythonclass InteractionSignatureCL(nn.Module):
    """
    交互签名对比学习 (Interaction Signature Contrastive Learning)
    创新点：不对比原始特征，而是对比正交特征的 Hadamard 交互模式。
    """
    def __init__(self, temperature=0.1):
        super().__init__()
        self.temperature = temperature

    def forward(self, V_g, V_c_perp, labels):
        # 1. 计算交互签名 (Hadamard 积)
        # 物理意义：提取基因和化学纯净特征之间的非线性协同激活模式
        H_sig = V_g * V_c_perp
        H_sig = F.normalize(H_sig, dim=-1) # 将签名投影到单位球面上

        # 2. 构建监督对比掩码 (Supervised Contrastive Mask)
        # 目标：让 Batch 内所有同类标签（同为1，或同为0）的样本签名互相靠近
        batch_size = labels.size(0)
        labels = labels.view(-1, 1)
        # pos_mask[i, j] = 1 表示 i 和 j 属于同一类（都是正样本，或都是负样本）
        pos_mask = torch.eq(labels, labels.T).float()
        
        # 排除自己对自己的计算
        mask_no_diag = 1.0 - torch.eye(batch_size, device=labels.device)
        pos_mask = pos_mask * mask_no_diag

        # 3. 计算所有交互签名两两之间的余弦相似度
        sim_matrix = torch.matmul(H_sig, H_sig.T) / self.temperature

        # 4. 为了数值稳定性，减去每行的最大值 (防止 softmax 溢出)
        max_sim, _ = torch.max(sim_matrix, dim=1, keepdim=True)
        logits = sim_matrix - max_sim
        exp_sim = torch.exp(logits) * mask_no_diag

        # 5. 计算 InfoNCE 变体的对比损失
        # 分母：所有其他样本的相似度指数和
        denom = exp_sim.sum(dim=1, keepdim=True) + 1e-8
        
        # 分子逻辑：计算同类样本的对数似然
        log_prob = logits - torch.log(denom)
        
        # 只有在 batch 内有同类别的其他样本时才进行惩罚
        valid_pos_count = pos_mask.sum(dim=1)
        valid_mask = valid_pos_count > 0

        # 求和取平均
        loss = - (pos_mask[valid_mask] * log_prob[valid_mask]).sum(dim=1) / valid_pos_count[valid_mask]

        return loss.mean() if valid_mask.any() else torch.tensor(0.0, device=labels.device)
