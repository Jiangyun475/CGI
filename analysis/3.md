三、 纯属创新的新型 CL 构想：正交特异性对齐 (Orthogonal Specificity Alignment, OSA)结合你的批判哲学：“不要把所有正样本拉近（机制不同），不要把所有负样本拉近（失效原因不同），还要解决 15% vs 85% 的严重不平衡”。我们要设计一个完全只针对“特异性（Specificity）”进行优化的局域对比学习。1. 核心哲学重构：锁与钥匙的“唯一匹配性”经过 Ortho 剥离后，$V_{c\perp}$ 是剔除了基础化学背景后，这把药物最纯粹的**“特异性钥匙”；$V_g$ 是“锁”**。如果 Drug A 结合了 Gene A（Label=1），那么 $V_{c\perp}^A$ 必须是 $V_g^A$ 独一无二的钥匙。这意味着：$V_g^A$ 应该与自己的钥匙 $V_{c\perp}^A$ 高度匹配。$V_g^A$ 必须拒绝 Batch 内其他成功药物的钥匙（比如 $V_{c\perp}^B$）。2. 全新创新点：批次内正样本反事实交叉（Counterfactual InfoNCE on Positives Only）我们直接抛弃所有的负样本（Label=0），不在 CL 中计算它们。机制如下：从 Batch 中过滤出所有真实发生结合的样本（Label=1），假设有 $N$ 个。将这 $N$ 个基因 $V_g$ 和 $N$ 个化学特异性特征 $V_{c\perp}$ 提取出来。把它们当作 $N$ 对（Anchor-Positive）。计算 $N \times N$ 的相似度矩阵。对角线是真实的互作对（匹配），非对角线是反事实的互作对（把 A 基因和 B 药物的特异性特征强行拼在一起，这在生物学上必然是不匹配的）。施加 InfoNCE Loss。3. 为什么这个构想在逻辑上完美无瑕？生物学极度合理： 它完美解决了你提出的“强行聚类”谬误。OSA 不仅没有拉近不同的正样本，它反而通过 InfoNCE 的非对角线惩罚，把不同机制的正样本在空间中强行推开了！ 它告诉网络：“激酶的钥匙打不开 GPCR 的锁，每种互作机制都必须占据流形空间中独立的一角。”天然免疫类别不平衡： CL 模块内部根本不看 Label=0 的数据。BCE Loss 负责宏观的 0/1 分类；OSA 模块化身“正样本提纯专家”，专门负责在 15% 的正样本内部建立细粒度的特征特异性。物理意义与 Ortho 的绝佳配合： 如果没有 Ortho，用原始的 $V_c$ 做交叉匹配是错的（因为大家都可能有基础的碳骨架）。但有了 Ortho，$V_{c\perp}$ 是完全独立的特异性特征，用它做交叉排斥在数学上是极其严密的。四、 代码实现：正交特异性对比损失 (OSA-CL)这个模块只有 30 行代码，没有任何外部依赖，完全符合你的 train_summean.py 架构。你可以直接新建这个类，并在训练脚本中替换掉旧的 CL。Pythonclass OrthogonalSpecificityCL(nn.Module):
    """
    正交特异性对比学习 (Orthogonal Specificity Alignment)
    创新点：放弃负样本聚类，仅在正样本内部通过“反事实交叉”建立锁钥特异性。
    """
    def __init__(self, dim=128, temperature=0.07):
        super().__init__()
        self.temperature = temperature
        # 独立的浅层投影空间，防止 CL 梯度过度干扰主分类器
        self.proj_g = nn.Sequential(nn.Linear(dim, dim), nn.ReLU(), nn.Linear(dim, dim))
        self.proj_c = nn.Sequential(nn.Linear(dim, dim), nn.ReLU(), nn.Linear(dim, dim))

    def forward(self, V_g, V_c_perp, labels):
        # 1. 过滤：仅仅提取当前 Batch 中的正样本 (Label == 1)
        pos_mask = (labels == 1.0).bool()
        
        # 如果当前 batch 正样本少于 2 个，无法构成对比对，直接跳过
        if pos_mask.sum() < 2:
            return torch.tensor(0.0, device=labels.device, requires_grad=True)

        # 2. 获取正样本的基因与化学正交特征
        g_pos = V_g[pos_mask]
        c_pos = V_c_perp[pos_mask]

        # 3. 映射到对比空间并进行 L2 归一化 (标准的球状流形投影)
        z_g = F.normalize(self.proj_g(g_pos), dim=-1)
        z_c = F.normalize(self.proj_c(c_pos), dim=-1)

        # 4. 计算 N x N 的相似度矩阵 (N 为当前 batch 的正样本数)
        # 对角线：匹配的真实互作 (应该大)
        # 非对角线：反事实的错误互作 (应该小)
        sim_matrix = torch.matmul(z_g, z_c.T) / self.temperature

        # 5. 构建自监督标签：对角线为目标 (0, 1, 2, ..., N-1)
        N = z_g.size(0)
        target = torch.arange(N, device=labels.device)

        # 6. 对称 InfoNCE 损失 
        # Loss_G: 以基因视角，在一个真实的化学特征和 N-1 个错误的化学特征中找真实的
        loss_g = F.cross_entropy(sim_matrix, target)
        # Loss_C: 以化学视角，在一个真实的基因和 N-1 个错误的基因中找真实的
        loss_c = F.cross_entropy(sim_matrix.T, target)

        return (loss_g + loss_c) / 2.0
