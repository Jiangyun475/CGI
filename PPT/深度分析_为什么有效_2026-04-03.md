# 深度思考：为什么这个策略有效？效果上限在哪里？

> 核心问题：看起来简单的策略为什么效果可以？效果好坏的根本原因是什么？
> 是图模型？参数量？还是别的？
> 对比学习为什么有用？为什么没有断档提升？

---

## 一、先看数据的本质结构

**MCF7 原始数据统计**（其他细胞系类似）：

```
总样本数:    206,685
唯一药物数:   11,597
唯一基因数:      978
理论全组合:  11,341,866
实际覆盖率:      1.8%   ← 极度稀疏
每药物配对基因: 17.8 个（中位数 13）
每基因配对药物: 211 个

接近决策边界 (|z| ∈ [2.0, 2.5]) 样本: 45.9%  ← 几乎一半是噪声区
正样本率: 49.9%（几乎完全平衡）
```

这三个数字是理解整个项目的关键：
- **覆盖率 1.8%**：大量(drug, gene)组合根本没有实测数据，标签本身就不完整
- **45.9% 处于边界区**：快一半的正/负标签是由 |z|=2.0 阈值硬切割出来的，本质是软标签被强行二值化
- **每药物只配对 18 个基因**：在 Chemical Cold Split 下，测试集药物对模型来说是完全陌生的，只靠化学结构泛化

## 二、为什么这个策略"看起来简单"但效果可以？

### 2.1 任务的本质比看起来容易

表面上是预测药物-基因相互作用，实际上数据有很强的内在规律：

**规律1：基因的"易激活性"是强先验**

部分基因对几乎所有药物都响应（正样本率>0.7），另一些几乎不响应（<0.3）：
```
正样本率 > 0.7 的基因: 270/978 (27.6%)
正样本率 < 0.3 的基因: 264/978 (27.0%)
```
这意味着：**单纯靠"这个基因本来就容易被激活"就能解释相当一部分的正样本**。

基因序列编码器学到的很可能不只是序列特征，而是"哪些基因在MCF7这个环境下容易被扰动"——这个知识可以从训练集药物中学到，并泛化到测试集。

**规律2：药物的"广谱性"是另一个先验**

```
正样本率 > 0.7 的药物: 17.3%（这类药物几乎打什么基因都有效）
正样本率 < 0.3 的药物: 12.7%（这类药物基本无效）
中间的不确定区域:   70%
```
对于极端药物（全有效/全无效），模型几乎不需要关注基因信息，只需要判断药物类型。对中间的 70% 才是真正的挑战。

**规律3：Chemical Cold Split 没有想象的那么难**

每个测试药物只配对 13 个左右的基因（中位数）。这意味着测试集的"难度"是分散的——模型不需要对每个测试药物全部预测对，只需要在 13 个配对中做出合理判断。

### 2.2 图模型 vs 参数量：谁更重要？

**实验证据**：

| 方法 | AUC (MCF7) | 特征类型 |
|------|-----------|---------|
| RF（128+4096维手工特征） | 0.8597 | Morgan FP + 6-mer |
| Morgan FP（GIN框架，固定指纹） | 0.8710 | ECFP4 2048-bit |
| DrugOp（GIN端到端） | 0.8918 | 学习的图特征 |

**结论：GIN 比 Morgan FP 提升 +0.021，比 RF 提升 +0.032。**

关键问题：这个提升来自图结构，还是来自端到端训练？

**分析**：Morgan FP 也是图结构派生的（ECFP 本质是 message passing 的特例），但它是固定的，不随任务优化。GIN 的优势不是"图结构本身"，而是**端到端学习了任务相关的子结构权重**。

Chemical Cold Split 下，GIN 的优势特别明显，因为：
- Morgan FP 把所有子结构平等对待（2048 个 bit 独立）
- GIN 通过 BCE 梯度学会了"哪些子结构对 CGI 预测重要"
- 学到的子结构权重具有归纳偏置：相似局部结构 → 相似表示 → 相似预测

**参数量呢？**

RF 本质上参数量无限（树的结构是数据决定的）。DL Baseline（0.8822）和 DrugOp（0.8918）参数量差不多但差距 +0.01。参数量不是核心因素，**归纳偏置（图结构先验 + 正交正则）才是**。

### 2.3 Operator 范式的真正贡献

**不是 AUC，而是分解方式。**

DrugOp 和 SumMean 在 MCF7 上 AUC 完全相同（0.8923）。但 DrugOp 的结构强制：
```
Δh = Σ_k  spectrum_k × u_k
```
每个 u_k 必须正交，每个 spectrum_k 对应一种独立的互作模式。

这是一个**有约束的表示学习**——限制了模型能表达的函数空间，反而有助于泛化（正则化效果）。同时提供了可解释性（各模式可以对应生物学通路）。

本质上，DrugOp 是一种**结构化瓶颈**：强制把药物-基因互作分解成 r=8 个独立维度，牺牲极小量的表达能力，换取可解释性和正则化收益。

## 三、对比学习为什么有用？深层原因

### 3.1 数据对比

| 方法 | MCF7 | A549 | VCAP | 与 wo_CL 差 |
|------|------|------|------|------------|
| SumMean（含CL） | 0.8923±0.0026 | 0.8635±0.0045 | 0.8151±0.0046 | 基准 |
| wo_CL（去掉CL） | 0.8907±0.0025 | 0.8611±0.0040 | 0.8115±0.0058 | −0.0016/−0.0024/−0.0036 |

**3/3 细胞系一致提升，且 VCAP 提升最大。**

### 3.2 表面解释

"CL 提供了额外的监督信号，帮助学习更好的表示。"

### 3.3 深层原因

**为什么 CL 在 Chemical Cold Split 下特别有用？**

**核心问题**：在只有 BCE 损失的情况下，药物编码器只通过"我预测对了多少药物-基因对"来更新。这意味着：
- 如果药物 A 只见过 18 个基因，它的表示完全由这 18 个信号塑造
- 测试时碰到一个新药物，模型必须靠化学结构泛化

CL（这里指 GCAA 或类似的对比目标）在 **药物表示空间** 加入了额外约束：

```
"对同一个基因表达正样本反应的两个药物，它们的表示应该相似"
```

这等价于：**把药物按生物学功能分组，而不是仅仅按预测正确率优化**。

**更深的原因**：CL 解决了 BCE 的一个根本缺陷。

BCE 只关心单个预测对错：`L = -y·log(p) - (1-y)·log(1-p)`。

它不区分：
1. 药物 A 和药物 B 都对基因 G 有效，但预测原因完全不同
2. 药物 A 因为学到了"正确的化学特征"而预测对
3. 药物 C 因为过拟合训练集的偶然相关性而预测对

CL 强制：**如果两个药物都对基因 G 有效，它们的表示要相似**。这是一种先验：有相同效果的药物应该有相似的内部表示（药效团层面的一致性）。

**为什么 VCAP 提升最大（+0.036）？**

VCAP（AR 扩增，228K样本）是最难的任务。样本量大意味着：
- 更多的药物参与对比学习，正样本对更多
- CL 的统计力更强，能建立更可靠的"功能相似药物"群落
- 正交正则在 VCAP 上也最有效——这两个效果相互增强

### 3.4 为什么 CL 在我们的框架里效果有限（+0.001~+0.004）？

**因为 Chemical Cold Split 本身限制了 CL 的作用空间。**

CL 建立的是"已见药物"的对比关系。测试集的药物完全未见，CL 学到的药物相似性规律必须通过 **化学结构归纳** 才能迁移到测试集。

如果测试药物的化学骨架在训练集里根本没有近邻，CL 的帮助很有限。这是一个根本性的限制。

**理论上 CL 的上限**：如果训练/测试药物有充分的化学覆盖度，CL 可以大幅提升泛化（可能 +0.01~0.02）。在 Chemical Cold Split 的严格设定下，当前提升 +0.002 左右是合理的。

## 四、为什么没有断档提升？——效果天花板分析

### 4.1 数据天花板（最根本原因）

**45.9% 的样本处于边界区（|z| ∈ [2.0, 2.5]）**

这意味着：
- 这些样本的标签是 "2.01 是正，1.99 是负" 的硬切割
- 重复实验可能导致这些标签反转
- **任何模型在这些样本上的最优策略都是随机猜**

理论最高 AUC 估算：假设边界区样本（46%）完全随机，其余（54%）完全预测正确：
```
理论上界 AUC ≈ 0.54 × 1.0 + 0.46 × 0.5 = 0.77... 
```
等等这不对，AUC 的计算不是线性的。更准确地说：

边界区样本的噪声会引入约 `0.46 × 0.25 ≈ 0.115` 的 AUC 损失（25% 是随机猜时的方差贡献）。当前模型 AUC ≈ 0.89，理论上界估计在 **0.92~0.95**。

**结论：我们已经接近当前数据质量下的理论上界了。**

### 4.2 任务本身的信息不足

L1000 只测量 978 个 landmark 基因（整个转录组有 ~20000 个基因）。药物效果是全转录组层面的，但我们只能看 4.9% 的信号。

同时，我们丢失了：
- **浓度信息**（LINCS 测的是单一浓度，而 CGI 是浓度依赖的）
- **时间信息**（固定 6/24h，而药物动力学是时变的）
- **细胞状态信息**（同一细胞系的不同实验批次、传代数）

### 4.3 模型架构已经足够简单

当模型足够简单但已经有效，增加复杂性提升有限。当前 918K 参数，对 206K 样本来说已经是合理的参数-数据比。

深度学习的"scaling law"在这里不适用：增加参数不会突破数据噪声造成的天花板。

## 五、还能怎么刷指标？——可行性排序

### 5.1 高收益（预期 +0.01~+0.02）

**A. 软标签（Label Smoothing with z-score）**

把二值标签替换为连续软标签：
```python
soft_label = sigmoid((zscore - 2.0) / 0.5)  # 正样本
# 或者
soft_label = zscore_normalized  # 归一化到 [0,1]
```
**理由**：消除边界区 45.9% 样本的噪声。z=3.0 和 z=2.01 都被当作 label=1 训练，前者应该得到更强的监督。

**风险**：会改变评估协议（但推理时仍用二值判断），需要重新设计 loss 和推理头。

**B. 多细胞系联合训练（Multi-task Learning）**

同一个 GIN 编码器，4个细胞系共享，4个独立的算子头。
```
Drug → GIN → h_drug
Gene → CNN → h_gene (共享或独立)
h_drug → Operator_MCF7 → spectrum_MCF7 → pred_MCF7
h_drug → Operator_A375 → spectrum_A375 → pred_A375
...
```
**理由**：药物的化学特征跨细胞系共享，4倍的训练数据 → 更好的药物表示泛化。VCAP 预期提升最大（因为 VCAP 单独训练时数据最"稀疏"）。

**预期效果**：+0.01~+0.02，尤其 VCAP/A549。

### 5.2 中等收益（预期 +0.005~+0.01）

**C. 5-fold Ensemble**

把5个 fold 的模型做 soft-vote 集成（对测试集取 5 个模型预测概率的均值）。
- 不需要额外训练，模型已经有了
- 理论增益：~+0.003~0.008（经验值）
- 注意：Chemical Cold Split 下 fold 之间测试集不同，需要用 out-of-fold 预测

**D. 置信度加权 BCE（Confidence-Weighted Loss）**

按 |z-score| 给每个样本赋予权重：
```python
weight = (|zscore| - 2.0).clamp(min=0.0)  # 边界区 weight→0
```
直觉：z=5.0 的正样本应该比 z=2.1 的正样本得到更强的惩罚。

**E. 加入细胞系基础表达量**

CCLE 数据库有各细胞系的基础基因表达谱（mRNA-seq）。把细胞系的基础表达量作为额外特征加入：
```
h_gene_context = CNN(gene_seq) + Linear(basal_expression)
```
这提供了"在这个细胞系里，这个基因通常的表达水平"，是模型现在完全缺失的信息。

### 5.3 低收益但值得尝试（+0.002~0.005）

**F. 更大的算子秩 r=16**

目前 r=8，增加到 r=16 让模型有更多的互作维度。
- 参数量增加约 30%
- VCAP（最难任务）可能受益

**G. SMILES 数据增强**

同一分子有多种 SMILES 表示（随机化枚举），用于训练增强：
- 可能帮助 GIN 学到更鲁棒的局部结构特征
- 实现简单（RDKit 支持随机化 SMILES）

### 5.4 不建议（高风险或违背设计原则）

| 方法 | 原因 |
|------|------|
| 预训练分子模型（ChemBERTa等） | 用户明确不要预训练；且违背端到端可解释性原则 |
| 更深的GIN（>3层） | L1000 分子多为中等复杂度，过深容易过平滑 |
| Transformer 替代 GIN | 参数大幅增加但数据没有增加 |
| 基于 protein 结构的特征 | 978个基因的 3D 结构数据难以获取且引入新的数据源 |

## 六、总结：核心回答

### Q1：为什么简单策略效果也可以？
因为任务本身有隐含规律：**基因的"易激活性"和药物的"广谱性"是强先验**，模型很大程度上在学这两个先验的组合，不需要精细的分子-序列交互推理就能达到约 0.85 AUC。

### Q2：效果好坏的核心原因是什么？
**不是参数量，不是架构复杂度，而是归纳偏置的质量**：
- GIN 的图结构归纳偏置（局部子结构 → 全局性质）✅
- 正交正则的表示分离偏置（8个独立模式）✅
- Chemical Cold Split 的泛化挑战定义了真正的上界

### Q3：CL 为什么有用？
**CL 在药物表示空间施加了"有相同生物效果的药物应该相似"的约束**，这是 BCE 无法提供的先验。它实质上是一种**基于生物功能的表示正则化**，在 Chemical Cold Split 下通过结构归纳帮助泛化。VCAP 受益最大因为其样本量最大，对比对更充分。

### Q4：为什么没有断档提升？
**数据天花板**：45.9% 样本处于 |z| ∈ [2.0, 2.5] 的噪声区，标签本身不稳定。当前 AUC≈0.89 已接近当前数据质量下的理论上界（估计 0.92~0.95）。增加模型复杂度无法突破数据噪声。

### Q5：可以怎么刷指标？
**优先级**：
1. **软标签**（用 z-score 替代硬标签）：预期 +0.01~0.02，解决根本问题
2. **多细胞系联合训练**：预期 +0.01~0.02，扩大有效训练数据
3. **5-fold Ensemble**：+0.003~0.008，免费的提升
4. **细胞系基础表达量**：中等收益，提供现在缺失的细胞背景信息

## 七、对论文写作的启示

这个分析说明，我们不应该把性能提升作为论文的核心卖点（因为天花板客观存在），而应该把重点放在：

1. **可解释性的独特性**：同等性能下，我们的方法提供了其他方法没有的细胞系特异性互作模式（VCAP 用 M4/M6/M7，MCF7 用 M2/M3）
2. **化学冷切分的严格性**：我们的评估比很多工作更严格，AUC 0.89 的含义更强
3. **端到端的内在一致性**：从化学结构到基因序列到互作谱，所有层次统一优化，没有黑盒
4. **数据天花板的论证**：边界区 45.9% 的分析本身就是一个有价值的发现，可以作为 Discussion 的一部分——说明 L1000 数据质量的内在限制，为社区提供参考
