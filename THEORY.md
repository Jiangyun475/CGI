## 1. 我们在解决什么问题？

**任务定义（一句话）：**
> 给定一个化学品（药物）和一个基因，预测：这个化学品能不能显著改变这个基因的表达量？

这个预测叫做 **Chemical-Gene Interaction (CGI)** 预测。

**为什么重要？**

- 新药研发：知道哪些化合物能调控某个靶标基因，就能缩小候选药物范围
- 毒理评估：工业化学品接触后，哪些基因会受影响？对应什么组织毒性？
- 精准医疗：同一个药，在 MCF7（乳腺癌细胞）和 VCAP（前列腺癌细胞）里效果不同，为什么？

**我们的数据：**

- 来源：LINCS L1000 项目（全球最大药物-基因转录组数据库）
- 4 个人类细胞系：MCF7（乳腺癌/ER+）、A375（黑色素瘤/BRAF突变）、A549（肺腺癌/KRAS）、VCAP（前列腺癌/AR扩增）
- 每条样本 = 一个（化合物，基因）对，标注正/负
  - 正例：logFC > 1.5 且 q-value < 0.05（显著差异表达）
  - 负例：无显著变化
- 化学冷分割（Chemical Cold Split）：测试集的化合物在训练集中从未出现过
  - 这是最严格的评估方式，模型必须能泛化到"从未见过的分子"

## 2. 之前的方法有什么问题？

### 2.1 传统机器学习：固定指纹

最常见的做法：
```
药物 → Morgan 指纹（固定 2048-bit 向量）
基因 → k-mer 频率统计
拼接 → 随机森林 → 预测
```

**问题：** Morgan 指纹是固定的——不管什么任务，苯环就是那几个 bit，无法学习。
在化学冷分割场景下，模型看到新化合物，只能靠局部结构相似性猜测，泛化能力差。

**结果：** MCF7 AUC ≈ 0.87，比我们的方法低约 0.023。

### 2.2 早期深度学习：黑盒拼接

```
药物 → GNN → drug_embedding [H]
基因 → CNN → gene_embedding [H]
拼接 → MLP → 预测
```

**问题：**
- 预测结果无法解释："模型为什么预测这个药会影响这个基因？"
- drug_embedding 和 gene_embedding 是独立学习的，没有显式建模"药物作用于基因的机制"

### 2.3 对比学习方法（我们之前尝试过，失败了）

对比学习的思路（GCAA 等方法）：
```
让"上调同一基因的不同药物"的表示靠近（正对）
让"上调不同基因的药物"表示远离（负对）
```

**失败原因：**
- **表示坍缩（Representation Collapse）**：所有基因嵌入塌缩到同一个点，density=1.0
- 对比损失和 BCE 损失之间存在梯度冲突
- 结果：AUC 反降，远低于普通分类

这不是实现问题，而是根本原理问题：对比学习的"拉近正对"会让基因表示失去区分性。

## 3. 核心思想：把药物当成"算子"

### 3.1 直觉解释

传统方法把药物当成一个**静态标签**：
> "布洛芬 = [0.1, 0.3, 0.8, ...]"（固定的数字向量）

我们的想法：药物的本质是一种**扰动**——它对细胞基因组表达空间施加了一个"推力"。
不同的药效团（pharmacophore，即分子中负责生物活性的结构片段）各自推向不同的方向。

所以，我们把药物当成**扰动算子** T：
```
T = I + Σ_k σ_k · (u_k ⊗ v_k^T)
```

用大白话解释：
- `I`：没有药的基准状态（恒等算子，什么都不改变）
- `u_k`：第 k 个**药效团的方向**（药物方面，"这个结构片段的输出方向"）
- `v_k`：第 k 个**基因响应的方向**（基因方面，"这类基因会被怎样影响"）
- `σ_k`：这个模式的**强度**（正值=激活，负值=抑制，接近0=这个模式不参与）

整个算子 T 作用到基因表达向量 h_g 上：
```
T(h_g) = h_g + Σ_k σ_k · (v_k · h_g) · u_k
                              ↑                ↑
               这个基因在第k个模式上的响应    药效团的输出方向
```

**关键洞察**：
- 不同细胞系，`v_k` 学到的基因方向不同 → 自动捕获细胞系特异性
- `σ_k` 可正可负 → 自然解释上调（正）和下调（负）
- 8 个独立模式（r=8） → 8 类药效团-基因交互机制

### 3.2 "交互谱"是什么？

```python
spectrum_k = σ_k × (v_k · h_g)  # [B, r]
```

对于每一个（药物，基因）对，我们得到一个 8 维向量：
- `spectrum[0]`: 模式 0 的激活强度（正=此药效团与此基因强激活）
- `spectrum[3]`: 模式 3 的激活强度（负=此药效团抑制此基因响应方向3）
- ...

这个 8 维向量就是**交互谱**（Interaction Spectrum），相当于这个（药物, 基因）对的"机理指纹"。

**可解释性的价值：**
- 正样本的谱应该 > 负样本（实验验证）
- 同类靶标的药物，谱向量应该相似（可用 t-SNE 可视化聚类）
- 每个模式对应什么生物学意义？ → 药效团热图告诉你

## 4. 模型架构：四个模块

```
输入：药物分子图 + 基因序列

┌─────────────────────────────────────────────────────────────┐
│  模块①：基因编码器（GeneMultiHeadReader）                    │
│                                                              │
│  基因序列 (DNA/RNA) → k-mer tokenize（6-mer，4096词汇）     │
│       ↓                                                      │
│  Embedding（把每个6-mer映射到128维向量）                     │
│       ↓                                                      │
│  4种大小的CNN卷积（捕获6-12个碱基的局部模式）               │
│       ↓                                                      │
│  8个注意力头（Multi-Head Attention）                         │
│  每个头学一个"关注方向"：哪些碱基位置与第k个药效团最相关？   │
│  输出：h_g_modes [B, 8, 128]  ← 8个角度看同一条基因序列    │
└─────────────────────────────────────────────────────────────┘
         ↓                    ↑
         ↓                    ↑ 模式对齐（核心）
┌──────────────────┐    ┌──────────────────────────────────────┐
│ 模块②：药物编码器 │    │ 模块③：扰动算子（PerturbationOp）    │
│                  │    │                                        │
│ 分子图 → GIN×3   │    │ 对每个模式 k：                        │
│  - 3层图卷积     │    │   u_k = 药效团嵌入方向（来自模块②）   │
│  - 原子→邻居消息  │    │   v_k = 基因的第k个响应方向           │
│  - 学习原子嵌入  │    │                                        │
│       ↓          │    │   coupling_k = v_k · h_g_modes_k      │
│ 8个药效团提取器   │    │   (点积：两个方向的对齐程度)          │
│ （Cross-Attention）│   │                                        │
│ pharma_emb       │    │   σ_k = tanh(MLP(pharma_emb_k))      │
│  [B, 8, 128]    │    │   (学习这个模式的强度，有界[-1,1])     │
│                  │    │                                        │
│  ↑ 每个slot学一  │    │   spectrum_k = σ_k × coupling_k       │
│  个药效团片段    │    │   (最终的交互谱：强度×对齐)            │
└──────────────────┘    └──────────────────────────────────────┘
                                     ↓
┌─────────────────────────────────────────────────────────────┐
│  模块④：分类头                                               │
│                                                              │
│  delta_h = Σ_k spectrum_k × u_k  ← 药物净扰动向量           │
│  features = [h_g_global, delta_h]  ← 拼接基因+扰动          │
│       ↓                                                      │
│  MLP(256→128→1) + Sigmoid                                   │
│  → P(这对药物-基因相互作用 = 1)                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键设计细节

| 组件 | 设计选择 | 理由 |
|------|---------|------|
| 基因 k-mer | 6-mer（4096 词汇） | 捕获局部核苷酸模式，无需预训练 |
| CNN 多卷积核 | 6, 8, 10, 12 碱基 | 覆盖不同长度的调控元件 |
| GIN 图卷积 | 3 层，含边特征 | 原子+键类型一起学习 |
| 零初始化注意力 | attn_queries = 0 | 避免随机注意力破坏早期训练 |
| Warmup 5 epochs | lr/10 → lr | 给注意力参数时间从零开始学习 |
| 正交正则 | lam_ortho = 0.1 | 确保 8 个模式独立（见第7节） |

## 5. 与对比学习有什么不同？

这是一个关键区别，因为我们之前尝试过对比学习并放弃了。

### 对比学习（Contrastive Learning）

**代表方法**：SimCLR、GCAA、BioLinkBERT...

```
核心思想：学习一个表示空间，让"语义相似"的样本靠近
损失函数：InfoNCE = -log(正对相似度 / 所有对相似度之和)

在 CGI 场景：
  正对 = 调控同一基因的两个药物
  负对 = 调控不同基因的药物对
目标：让"效果相似的药"表示相似
```

**我们的方法（DrugOperatorNet）**

```
核心思想：直接预测（药物, 基因）对的相互作用
损失函数：BCE = -[y·log(p) + (1-y)·log(1-p)]

没有"对"的概念，每个样本独立预测
```

### 关键区别

| 对比 | 对比学习 | DrugOperatorNet |
|------|---------|-----------------|
| 训练目标 | 表示空间对齐 | 直接二分类 |
| 可解释单元 | 表示向量（抽象） | 交互谱（有物理意义） |
| 样本关系 | 需要构建正负对 | 独立预测每个样本 |
| 失败风险 | 表示坍缩（我们碰到过） | 梯度稳定 |
| 细胞系特异性 | 难以显式建模 | v_k 天然学习细胞系特异性方向 |
| 正则化目标 | 均匀分布（避免坍缩） | 正交性（模式独立） |

**一个直觉上的对比：**

- 对比学习 ≈ "把相似的东西放一起" — 地图上的聚类问题
- DrugOperatorNet ≈ "分析作用机制" — 力学分解问题（力=方向×强度）

## 6. 与之前版本有什么不同？

我们的代码演化经历了几个版本：

### 版本 0：BiasAware（已废弃）

```
drug_emb [H] + gene_emb [H] → 偏差感知融合 → 分类
```

问题：完全黑盒，偏差分支没有物理意义，AUC < 0.89

### 版本 1：MoE（混合专家）

```
[V_g, V_c⊥] → 路由器 → K个专家 → 加权分类
```

- K=4 个专家，路由信号是"基因方向"和"化学品方向的正交分量"
- AUC 0.8935，比 BiasAware 好
- 问题：**为什么是这 K 个专家？没有生物学意义**

### 版本 2：MoE + Targeted Pooling

```
路由权重 × 专家查询 → 原子注意力 → 靶向池化
```

- AUC 0.8943，最高的黑盒方法
- 问题：仍然没有可解释的中间量

### 版本 3：DrugOperatorNet V1（第一版）

第一次引入算子概念：
```
drug_emb [H] → U, V, σ → spectrum → delta_h
```

- 所有模式共用同一个 h_g（非模式对齐）
- AUC 0.8924，接近 MoE 但可解释

### 版本 4：DrugOperatorNet V2（当前主模型）

**关键改进**：模式对齐耦合

```
GeneMultiHeadReader → h_g_modes [B, r, H]  ← r个角度看基因
coupling_k = v_k · h_g_modes_k             ← 第k个药效团和第k个基因方向
```

V1 的问题：所有模式的基因响应共用同一个 h_g，模式之间不够独立
V2 的解决：每个模式有自己专属的基因视角 h_g_modes_k

- AUC 0.8923（与 V1 相近，但理论上更严格）

### 版本 5：OperatorMoE（消融实验用）

将 V2 和 MoE+Target 融合，设计了 4 种消融变体：

| 变体 | AUC | 结论 |
|------|-----|------|
| full（谱路由MoE） | 0.8868 | 最差：MoE梯度冲突 |
| no_spectrum | 0.8917 | 传统MoE路由 |
| **no_moe（主模型）** | **0.8923** | 最优：纯算子 |
| no_moe_noortho | 0.8915 | 正交有效(+0.0008) |

**结论：DrugOperatorNet（no_moe 配置）= 最优模型**

## 7. 正则化：为什么要让模式"彼此独立"？

### 问题背景

我们有 8 个药效团模式（u_1, u_2, ..., u_8）。
如果没有约束，模型很可能学到：

```
u_1 ≈ u_2 ≈ u_3 ≈ ... ← 所有模式都表示同一件事
```

这样就失去了"多模式分解"的意义，可解释性为零。

### 解决方案：正交正则化

我们要求 U 矩阵的列向量两两正交（就像 x, y, z 三个坐标轴彼此垂直）：

```python
U_n = normalize(U, dim=-1)           # 归一化
gram = U_n @ U_n.T                   # Gram 矩阵（测量两两相似度）
loss_ortho = ||gram - I||²           # 目标：接近单位矩阵（两两正交）
```

**消融验证：**
- lam_ortho=0.1：AUC 0.8923
- lam_ortho=0.0（去掉正则）：AUC 0.8915
- 差异 +0.0008，但更重要的是**可解释性价值**：
  - 有正交正则 → 8 个模式分工明确，相关矩阵接近单位矩阵
  - 无正交正则 → 模式互相冗余，无法解释哪个模式代表什么

## 8. 可解释性：模型能告诉我们什么？

这是 NMI 投稿的核心价值。我们能提取三层可解释信息：

### 层次 1：交互谱（Interaction Spectrum）

**是什么：** 每个（药物，基因）对的 8 维向量，表示 8 种机制的激活程度

**能做什么：**
- 正样本的谱幅度 > 负样本（待验证，见 `analyze/visualize_spectrum.py`）
- t-SNE 可视化：相同靶标的化合物在谱空间中应聚在一起
- 稀疏性：预期每对只激活 2-3 个主要模式

### 层次 2：药效团热图（Pharmacophore Heatmap）

**是什么：** 在分子图上，每个原子对每个模式的贡献权重

**能做什么：**
- 直接可视化"哪个原子最重要"（颜色越深=越重要）
- 验证：已知药效团（如芳香环、氢键供体）应该高亮
- 模式区分：模式1可能关注芳香性，模式2可能关注氢键...

**见：** `analyze/visualize_pharmacophore.py`

### 层次 3：基因注意力（Gene Attention）

**是什么：** 基因序列上，每个 k-mer 位置对每个模式的注意力权重

**能做什么：**
- 找到"响应某类药效团的关键调控元件"
- GO 富集：高注意力基因 → 对应什么信号通路？
- 细胞系特异性：MCF7 vs A375 的注意力模式差异

**见：** `analyze/visualize_gene_attention.py`

## 9. 这个工作够发 NMI 吗？差什么？

### NMI 的核心标准

Nature Machine Intelligence 要求：**机器学习推进自然科学发现**

不只是"AUC 高了 X%"，而是"因为 ML，我们学到了什么新的生物学知识？"

### 当前状态（√/✗）

| 要求 | 状态 | 说明 |
|------|------|------|
| 新颖的建模视角 | ✓ | 算子范式（非黑盒嵌入） |
| 优于现有方法 | ✓（部分） | 优于 Morgan FP，等于 MoE |
| 严格评估（冷分割） | ✓ | Chemical Cold Split |
| 多细胞系验证 | ✓ | 4 个细胞系 |
| 消融实验 | ✓ | 4 个变体 |
| SOTA 对比 | ✗ | **缺：DeepCE、DECIPHIR** |
| 5-fold CV 统计显著性 | ✗ | **缺：只有 fold0** |
| 谱分析（可解释性） | ✗ | **缺：代码有，图没生成** |
| 药效团热图 | ✗ | **缺：需要修改代码提取原子注意力** |
| 基因注意力+GO富集 | ✗ | **缺：生信分析未做** |
| 生物学发现 | ✗ | **缺：所有可解释性图都没有** |

### 优先级排序

```
P0（本周）：谱分析 + 药效团热图（代码已提供，直接运行）
P1（下周）：5-fold CV + SOTA 对比方法实现
P2（两周）：基因注意力 GO 富集
P3（一月）：论文全文
```

## 附录：数学符号对照表

| 符号 | 含义 | 维度 |
|------|------|------|
| B | batch size | 整数 |
| H | hidden dim（隐层维度） | 128 |
| r | operator rank（算子秩 = 药效团数 = 注意力头数） | 8 |
| L | 基因序列长度（k-mer） | 1000 |
| N | 一个 batch 中总原子数 | 变长 |
| h_g_modes | 基因的多视角编码 | [B, r, H] |
| h_g_global | 基因的全局表示 | [B, H] |
| pharma_emb | 药效团嵌入（r 个 slot） | [B, r, H] |
| U | 药效团输出方向（已归一化） | [B, r, H] |
| V | 药效团输入方向（已归一化） | [B, r, H] |
| σ | 模式强度（∈ [-1, 1]） | [B, r] |
| coupling | 药物-基因方向对齐度 | [B, r] |
| spectrum | 交互谱 = σ × coupling | [B, r] |
| delta_h | 净扰动向量 = Σ spectrum_k × u_k | [B, H] |
